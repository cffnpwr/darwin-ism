# CLAUDE.md - darwin-ism

## Project Summary

darwin-ism is a macOS CLI tool for managing input sources (keyboard layouts, input methods) using the Carbon Framework's TIS API. It enables programmatic listing, enabling, and disabling of input sources that macOS restricts via standard settings. Built with Swift 6.2 and Nix Flakes.

## Critical Rules

- SwiftLint runs in **strict mode** — all warnings are errors. Run `swiftlint lint --config .swiftlint.yaml --strict` before committing.
- Format with `treefmt` before committing. CI fails on unformatted code.
- The Swift version in `mise.toml` and `.swift-version` must match the version from `flake.lock`. The script `scripts/check-swift-version.sh` verifies this.
- The version string in `CLI.swift` is managed by release-please markers (`x-release-please-start-version` / `x-release-please-end`). Do not manually edit the version.
- `nix/darwin-ism/generated/` files are auto-generated by swiftpm2nix. Do not edit manually.

→ Details: `.agents/docs/critical/rules.md`

## Quick Start

```bash
# Enter dev shell (Nix)
nix develop

# Build
swift build

# Run
./.build/debug/darwin-ism list
./.build/debug/darwin-ism list --enabled
./.build/debug/darwin-ism enable <id>
./.build/debug/darwin-ism disable <id>

# Lint (strict)
swiftlint lint --config .swiftlint.yaml --strict

# Format
treefmt
```

→ Full reference: `.agents/docs/reference/commands.md`
