# Critical Rules

## Build and Quality Gates

All of the following must pass before committing:

1. **Build**: `swift build` must succeed without errors.
2. **Lint**: `swiftlint lint --config .swiftlint.yaml --strict` — strict mode treats warnings as errors.
3. **Format**: `treefmt --fail-on-change` — code must match formatting standards.

Pre-commit hooks (lefthook) enforce lint and format automatically.

## Version Consistency

The Swift version must be consistent across:
- `flake.lock` (source of truth, derived from nixpkgs)
- `mise.toml`
- `.swift-version`

Run `scripts/check-swift-version.sh` to verify. This is also checked by lefthook pre-commit.

## Auto-Generated Files

Do not manually edit:
- `nix/darwin-ism/generated/default.nix` — generated by swiftpm2nix
- `nix/darwin-ism/generated/workspace-state.json` — generated by swiftpm2nix

## Release-Please Markers

The version in `Sources/darwin-ism/CLI.swift` is managed by release-please:

```swift
// x-release-please-start-version
version: "0.1.0",
// x-release-please-end
```

Do not manually change the version string. It is updated automatically by the release-please workflow.

## CI Requirements

CI runs on every push to main and on PRs:
- **build**: Compiles on both arm64 (macOS-15) and amd64 (macOS-15-Intel)
- **swiftlint**: Strict mode
- **format**: Fails on any formatting difference
